# Sister Store Recommendation Engine

## Business Problem

Retail planning teams frequently need to identify comparable stores when evaluating assortment decisions, forecasting performance, or supporting stores with limited historical data. Manual identification of comparable stores can be time-consuming and inconsistent.

## Objective

Develop a recommendation framework that identifies similar stores based on geographic and operational characteristics.

## Approach

* Evaluated store attributes and geographic relationships
* Developed recommendation logic to identify comparable stores
* Leveraged location-based analytics and business rules to generate store pairings
* Validated recommendations with business stakeholders
* Created a scalable process that reduced manual analysis requirements

## Technologies

* SQL
* BigQuery
* Python
* Geospatial Analytics
* Recommendation Systems
* Retail Analytics

## Impact

* Reduced manual effort required to identify comparable stores
* Improved consistency of store pairing recommendations
* Supported assortment and planning decisions for stores with limited data
* Increased efficiency of retail planning workflows

## Key Learnings

Recommendation systems are not limited to customer-facing applications. Similarity-based approaches can provide substantial value in operational and planning processes by helping organizations make more informed decisions.
