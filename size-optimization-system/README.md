# Size Optimization System

## Business Problem

Retail inventory performance depends heavily on allocating the right sizes to the right stores. Traditional approaches often rely on historical assumptions that may not accurately reflect local customer demand, resulting in lost sales opportunities and excess inventory.

## Objective

Develop and maintain a size optimization framework that improves inventory allocation decisions by identifying the optimal size distribution for merchandise across stores while increasing operational efficiency for business users.

## Approach

* Analyzed historical sales and inventory data to identify size demand patterns
* Developed and maintained Buy Size Profiles (BSPs) used in allocation decisions
* Evaluated size performance across product categories and store locations
* Built automated workflows to support profile generation and review
* Created fallback methodologies for products with limited historical data
* Supported business stakeholders with profile validation and recommendation reviews
* Designed and developed a Python-based graphical user interface (GUI) that enabled non-technical users to generate and manage size profiles without writing code
* Automated the creation of bulk JSON files required for downstream API integrations, significantly reducing manual effort and onboarding complexity

## Technologies

* Python
* SQL
* BigQuery
* API Integration
* JSON
* GUI Development
* Retail Analytics
* Inventory Optimization
* Data Validation

## Impact

* Supported inventory allocation decisions across multiple merchandise categories
* Improved consistency and scalability of size profile generation
* Reduced manual effort required for profile creation and maintenance
* Accelerated onboarding for business users by providing a self-service interface
* Automated generation of large-scale JSON payloads used by downstream Size Optimization APIs
* Enabled broader adoption of optimization tools across technical and non-technical teams

## Key Learnings

Successful optimization solutions require more than strong analytical models. Creating user-friendly tools and automation workflows can dramatically increase adoption, reduce operational overhead, and make advanced analytics accessible to a wider audience.
