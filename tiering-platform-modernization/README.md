# Tiering Platform Modernization

## Business Problem

A legacy assortment tiering platform relied on more than 120 interconnected Microsoft Access queries to support merchandise planning and store-level assortment decisions. The existing solution faced scalability, maintainability, and performance challenges.

## Objective

Migrate the platform to a cloud-based architecture while preserving business logic, improving reliability, and reducing operational complexity.

## Approach

* Audited and documented existing business logic
* Translated over 120 Access queries into BigQuery SQL
* Validated outputs against legacy systems
* Refactored complex transformations for cloud execution
* Improved maintainability through standardized workflows
* Supported deployment within Google Cloud Platform

## Technologies

* SQL
* BigQuery
* Python
* Vertex AI
* Microsoft Access
* Google Cloud Platform

## Impact

* Modernized a critical retail decision-making platform
* Improved scalability and reliability
* Reduced query complexity and maintenance burden
* Supported assortment decisions across 500+ stores

## Key Learnings

Large-scale data modernization requires balancing technical improvements with business continuity. Successful migration depended on validating legacy outputs while creating a more scalable and maintainable architecture.
